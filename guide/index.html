What “residual leadership” means

NASDAQ-100 names often rise and fall together with the market (QQQ). We care about names that are stronger than QQQ over a multi-month window — the temporary “captain” of the index basket.

residual_return(day) ≈ stock_return(day) − QQQ_return(day)
score = cumulative residual over lookback, after a short skip (avoid last few noisy days)

Only names that were real index members on that decision date (point-in-time). No time-travel membership.

Engine math (implementation-level)

1) Daily residual

r_stock(t) = P(t)/P(t-1) − 1
r_qqq(t) = Q(t)/Q(t-1) − 1
resid(t) = r_stock(t) − r_qqq(t)

2) Residual strength score

cum(t) = product(1 + resid) from start
score(t) = cum(t − skip) / cum(t − lookback) − 1

Alpha One: lookback=84, skip=5. L3: lookback=126, skip=21, and require price momentum > 0 over same window.

3) Momentum filter (L3 only)

mom(t) = P(t − skip)/P(t − lookback) − 1
if mom ≤ 0: score = −∞ (cannot be picked)

4) Ranking & exit

  • Universe = NDX members on date t with price ≥ $5 and valid score.
  • Alpha One: hold if rank ≤ 2; else sell. Fill empty slot with best rank.
  • L3: every week, target = rank #1 only (force rotate).

5) Vol gear (L3 / Extreme)

vol(t) ≈ stdev(daily strategy returns, vol_lb) × √252
lev(t) = clip( scale × vol_tgt / vol(t−1) , 0, 6 )

scale is frozen (classic-era pin). Live bot uses lev at rebalance as target_weight.

6) Costs (research)

  • Unlever: ~10 bps adverse slip per side + IBKR-like commission + SEC/TAF on sells.
  • Lever: ~7 bps + same commissions + 5.5% APR on borrowed fraction max(lev−1,0).

Manual checklist (no Pine)

1

Universe

Use a current NDX list (TradingView “NASDAQ 100” components). Know this is not PIT-perfect.

2

Each rebalance day

Alpha One: last trading day of month. L3: last trading day of week (usually Friday).

3

Compute score per name

For each ticker, residual strength vs QQQ over lookback with skip (formulas above). Spreadsheet is fine.

4

Pick / hold

Alpha One: buy #1; hold while rank ≤ 2. L3: always hold current #1 with mom>0.

Pine Script v5 — educational approximation

This script plots residual strength of the chart symbol vs QQQ and marks rough monthly/weekly decision points. It does not scan all NDX or enforce PIT. Use on each candidate, or build a watchlist workflow.

//@version=5
// EDUCATIONAL ONLY — not PIT NDX scanner, not broker automation
indicator("Residual strength vs QQQ (approx)", overlay=false)

lookback = input.int(84, "Lookback", minval=20)
skip     = input.int(5, "Skip", minval=0)
useMom   = input.bool(false, "Require price mom > 0 (L3-style)")

qqq = request.security("QQQ", timeframe.period, close)
rStock = ta.roc(close, 1) / 100.0
rQqq   = ta.roc(qqq, 1) / 100.0
resid  = rStock - rQqq

// cumulative residual (log-sum style for stability)
cum = ta.cum(math.log(1.0 + nz(resid)))
// score ≈ growth of residual cum over [t-lookback, t-skip]
score = math.exp(cum[skip] - cum[lookback]) - 1.0
mom   = close[skip] / close[lookback] - 1.0
valid = not useMom or mom > 0
plot(valid ? score : na, "Residual score", color=color.new(color.aqua, 0))
hline(0, "Zero")

// decision markers (approx)
isMonthEnd = ta.change(time("M")) != 0
isWeekEnd  = ta.change(time("W")) != 0
plotshape(isMonthEnd, title="Month boundary", style=shape.triangledown, location=location.top, size=size.tiny, color=color.yellow)
plotshape(isWeekEnd, title="Week boundary", style=shape.circle, location=location.bottom, size=size.tiny, color=color.purple)

// Optional: simple vol-target gear on THIS symbol returns (not portfolio)
volLb  = input.int(42, "Vol LB")
volTgt = input.float(0.25, "Vol target ann.")
scale  = input.float(1.26, "Frozen scale (L3-like)")
ret    = ta.roc(close, 1) / 100.0
vol    = ta.stdev(ret, volLb) * math.sqrt(252)
lev    = vol > 0 ? math.min(6.0, math.max(0.0, scale * volTgt / vol)) : 1.0
plot(lev, "Approx lev", color=color.new(color.fuchsia, 0), display=display.pane)
How to use for “manual Next Medallion”: put NDX names in a watchlist → sort by residual score on rebalance day → Alpha One: take #1 if rank rules ok → L3: take #1 with mom filter → size with lev plot if you want.

Paper trade — full path (do this before live)

Goal: 4–12 weeks boring paper. Measure fills, Gateway uptime, schedule correctness. No live deposit required for paper.
!

Rules of engagement

DRY_RUN=1 until healthcheck + forced rebalance look correct. Never put IB password in git or chat. Kill file stops trading fast.

IBKR paper account

  1. Register at interactivebrokers.com → complete KYC for a real account (paper rides alongside).
  2. Account Management → open / reset Paper Trading account. Note paper account id (often starts with DU…).
  3. Market data: enable US equity data as required (paper may use delayed; confirm quotes work in Gateway).
  4. Permissions: US Stocks long. API: allow socket clients.
  5. Install IB Key 2FA on your phone.
Paper does not need deposited cash. Live does. Don’t confuse account numbers.

Provision a VPS (Windows recommended first)

  1. Pick provider (see VPS & money tab for SKUs/prices).
  2. Create Windows Server 2022 (or Win 10/11 cloud) · 2 vCPU · 4–8 GB RAM · 80 GB SSD.
  3. Region: Singapore OK for weekly/monthly; US East nicer for market hours comfort.
  4. Set admin password, enable RDP, restrict RDP to your IP in firewall.
  5. RDP in → install Windows updates carefully (avoid surprise reboot on rebalance day).
  6. Create folder C:\algo\.

Install & login IB Gateway (paper)

  1. Download IB Gateway (not full TWS if you want lighter) from IBKR.
  2. Login with paper credentials / select paper account.
  3. Configure → Settings → API:
    • Enable ActiveX and Socket Clients = ON
    • Read-Only API = OFF
    • Socket port = 4002 (Gateway paper default)
    • Trusted IPs: 127.0.0.1
    • Download open orders on connect = ON
  4. Leave Gateway running. Approve 2FA on phone when prompted.
  5. Optional later: IBC (IBController) for auto-restart login (advanced).
Gateway paper API  → 127.0.0.1:4002
TWS paper API       → often 7497
Live Gateway        → often 4001
Bot must use the same host/port as YOUR Gateway.

Go-live sequence

1

Freeze config

Pick ONE sleeve to automate first (suggest Alpha One unlever OR L3 Sleeve — not Extreme first). Write the frozen params in a note.

2

Funding

Deposit only what you can see draw down by the sim DD (~45–60%). Keep buffer for margin if levered.

3

Live Gateway

Separate login mode, port 4001 (confirm on your install), Read-Only OFF, trusted 127.0.0.1 only.

4

New .env live

Copy paper .env → .env.live with live account + port. Keep paper env intact for parallel paper if you want.

5

DRY_RUN=1 on live port

Connect + healthcheck against live Gateway without orders. Confirm account equity reads correctly.

6

Micro live order

DRY_RUN=0 with tiny MAX_GROSS (e.g. 0.1) once. Verify fill, commission, position.

7

Raise size slowly

Only after 2–4 clean rebalance cycles. Extreme sleeve last, if ever.

8

Monitoring

Telegram alerts optional; weekly human check of Gateway + logs; monthly reconcile vs broker statements.

Risk controls you must keep

  • state/KILL file — create empty file to halt new orders (confirm your bot honors it — ours does).
  • MAX_GROSS < 1 for cash buffer (e.g. 0.95–0.97 unlever; lever uses target_weight separately).
  • Never open Gateway API to the public internet.
  • One strategy per account if possible (A1 vs L3).
  • Tax / local regulation = your responsibility.
